    Overview
    This PR implements an automated CI/CD pipeline for deploying our backend to AWS App Runner. It improves our deployment reliability by adding proper health checks, enhancing error handling, and standardizing our deployment process through GitHub Actions.
    Changes

    🔄 Add GitHub Actions workflow for automated CI/CD
    🐳 Update Dockerfile for better compatibility with CI/CD process
    🔌 Enhance Supabase client with improved error handling
    🩺 Configure HTTP health checks for App Runner on the /health endpoint
    🔐 Set up secure environment variable configuration
    ✅ Implement deployment verification steps

    Technical Details

    The workflow automatically builds and pushes Docker images to ECR
    It intelligently creates or updates the App Runner service as needed
    Health checks now use HTTP protocol instead of TCP for more reliable validation
    Supabase client now includes better validation and error messages
    Deployment verification ensures the application is truly running

    Testing

    Successfully tested the workflow on a test branch
    Verified that health checks are correctly configured and functioning
    Confirmed proper environment variable passing

    Deployment Notes

    This PR requires the following GitHub secrets to be configured:

    AWS_ROLE_ARN
    APP_RUNNER_ACCESS_ROLE_ARN
    APP_RUNNER_AUTOSCALING_CONFIG_ARN
    SUPABASE_URL
    SUPABASE_KEY
    JWT_SECRET
    ALLOWED_ORIGINS
